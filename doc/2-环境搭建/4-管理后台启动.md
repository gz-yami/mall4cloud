# 管理后台启动

平台端和商家端都是 Vite 工程，命令和环境变量以当前 `package.json`、`.env.*` 为准。

## 安装依赖

平台端：

```bash
cd front-end/mall4cloud-platform
pnpm install
pnpm dev
```

商家端：

```bash
cd front-end/mall4cloud-multishop
pnpm install
pnpm dev
```

当前 Vite dev server 端口是 `9527`。如果两个后台同时启动，第二个需要手动换端口或按 Vite 提示处理。

## 关键环境变量

`.env.development` 和 `.env.production` 使用：

```text
VITE_APP_BASE_API
VITE_APP_RESOURCES_URL
VITE_APP_RESOURCES_TYPE
```

本地常见配置：

```text
VITE_APP_BASE_API = 'http://<网关IP>:8000'
VITE_APP_RESOURCES_URL = 'http://<MinIO-IP>:9000/mall4cloud'
VITE_APP_RESOURCES_TYPE = '1'
```

不要使用 `VUE_APP_*`，当前后台只读取 `VITE_APP_*`。

## 登录

默认账号参考 [数据库初始化](./2-数据库初始化.md#默认账号)：

- 平台端：`admin / 123456`
- 商家端：`admin / 123456`

登录失败先看浏览器 Network：

- 请求是否发到 `VITE_APP_BASE_API`。
- 是否返回 `A00004` 未授权。
- 是否跨域失败。
- auth、rbac、platform/multishop 服务是否已启动。

## 检查题

问题：图片能上传但前端显示 404，先改哪个配置？

答案：先核对 `VITE_APP_RESOURCES_URL` 和 Nacos 中 `biz.oss.resources-url` 是否都指向 `http://<MinIO-IP>:9000/mall4cloud`。

为什么：上传和展示使用的资源域名必须和 MinIO 桶地址一致，后台返回的是对象路径，前端需要拼接正确资源地址。
