# 环境要求

以下版本来自当前仓库配置。后台前端是 Vue 3 + Vite，后端要求 JDK 17。

## 后端

| 组件 | 当前要求 | 依据 |
| --- | --- | --- |
| JDK | 17 | 根 `pom.xml` 的 `java.version` |
| Maven | 3.9+ | 当前多模块 Maven 工程 |
| Spring Boot | 4.0.3 | 根 `pom.xml` |
| Spring Cloud | 2025.1.1 | 根 `pom.xml` |
| Spring Cloud Alibaba | 2025.1.0.0 | 根 `pom.xml` |
| Nacos Client | 3.1.1 | 根 `pom.xml` |
| Seata | 2.6.0 | 根 `pom.xml` |

## 前端

| 组件 | 当前要求 | 依据 |
| --- | --- | --- |
| Node.js | `^20.19.0 || >=22.12.0` | 平台端/商家端 `package.json` |
| pnpm | `>=8 <11` | 平台端/商家端 `package.json` |
| Vite | 8.x | 平台端/商家端 `package.json` |
| Vue | 3.5.x | 平台端/商家端 `package.json` |

前端包管理器必须使用 pnpm。`package.json` 里有 `preinstall: npx only-allow pnpm`，用 npm/yarn 安装会被拦截。

| Redis | 7.0 | Docker Compose |
| Nacos | 3.1.1 | Docker Compose |
| Seata | 2.6.0 | Docker Compose |

## 检查题

问题：当前后台前端启动前，应该先安装 pnpm 还是直接用 npm install？

答案：先安装并使用 pnpm。

为什么：平台端和商家端 `package.json` 声明了 pnpm 版本范围，并用 `only-allow pnpm` 限制安装工具。

问题：当前后端是否还是 JDK 8 项目？

答案：不是，当前要求是 JDK 17。

为什么：根 `pom.xml` 的 `java.version` 是 `17`。
