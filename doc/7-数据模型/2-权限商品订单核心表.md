# 权限、商品、订单核心表速查

本页只讲启动和二开最常用的表。

## 权限表

| 表 | 作用 | 关键字段 |
| --- | --- | --- |
| `auth_account` | 统一账号 | `username`、`password`、`sys_type`、`user_id`、`tenant_id`、`is_admin` |
| `menu` | 后台菜单和动态路由 | `menu_id`、`parent_id`、`biz_type`、`path`、`component` |
| `menu_permission` | 按钮和接口权限 | `permission`、`uri`、`method`、`menu_id`、`biz_type` |
| `role` | 角色 | `role_id`、`biz_type`、`tenant_id` |
| `role_menu` | 角色拥有的菜单和按钮 | `role_id`、`menu_id`、`menu_permission_id` |
| `user_role` | 用户拥有的角色 | `user_id`、`role_id` |

`sys_type` 和 `biz_type` 的常见含义：`1` 商家端，`2` 平台端。

## 商品表

| 表 | 作用 | 关键字段 |
| --- | --- | --- |
| `spu` | 商品主信息 | `spu_id`、`brand_id`、`category_id`、`shop_id`、`name`、`status` |
| `sku` | 具体销售 SKU | `sku_id`、`spu_id`、`sku_name`、`attrs`、`price_fee`、`status` |
| `sku_stock` | SKU 库存 | `sku_id`、`actual_stock`、`lock_stock`、`stock` |
| `sku_stock_lock` | 下单库存锁定 | `spu_id`、`sku_id`、`order_id`、`status`、`count` |
| `spu_detail` | 商品详情 | `spu_id`、`detail` |
| `spu_sku_attr_value` | SKU 销售属性 | `spu_id`、`sku_id`、`attr_name`、`attr_value_name` |

价格字段如 `price_fee` 用整数保存，二开时不要随意改成浮点金额。

## 订单表

| 表 | 作用 | 关键字段 |
| --- | --- | --- |
| `order` | 订单主表 | `order_id`、`shop_id`、`user_id`、`total`、`status`、`is_payed` |
| `order_item` | 订单商品明细 | `order_id`、`spu_id`、`sku_id`、`count`、`price`、`spu_total_amount` |
| `order_addr` | 收货地址快照 | `order_addr_id`、`user_id`、`consignee`、`mobile`、`addr` |
| `pay_info` | 支付记录 | `pay_id`、`user_id`、`pay_amount`、`pay_status`、`order_ids`、`version` |

订单状态当前 SQL 注释中说明：`1` 待付款，`2` 待发货，`3` 待收货，`5` 成功，`6` 失败。

订单主表中以下字段在读订单状态时需要一起看：

| 字段 | 含义 |
| --- | --- |
| `close_type` | 订单关闭原因，SQL 注释含 `1` 超时未支付、`4` 买家取消、`15` 货到付款交易 |
| `delete_status` | 用户侧删除状态：`0` 未删除、`1` 回收站、`2` 永久删除 |
| `version` | 订单版本号，SQL 注释为每处理一次订单版本号加一 |
| `order_addr_id` | 指向 `order_addr`，保存下单时的收货地址快照 |

## 关系速记

- 一个账号 `auth_account` 通过 `sys_type + user_id` 指向平台用户或商家用户。
- 一个用户通过 `user_role` 拥有多个角色。
- 一个角色通过 `role_menu` 拥有菜单和按钮权限。
- 一个商品 `spu` 有多个 `sku`，每个 `sku` 对应库存。
- 一个订单 `order` 有多个 `order_item`，每个明细记录当时的 `spu_id`、`sku_id`、价格和数量。
- 一个订单保存一个 `order_addr_id`，避免用户后续修改地址影响已下单记录展示。

## 检查题

问题：前端按钮权限编码应该去哪个表找？

答案：`mall4cloud_rbac.menu_permission.permission`。

为什么：前端 `v-permission` 使用的就是这个编码，后端接口权限也用它匹配用户权限。
