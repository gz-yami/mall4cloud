# 前端工程与动态路由

平台端和商家端是 Vue 3 + Vite + Pinia + Element Plus，环境变量使用 `VITE_APP_*`。

## 关键目录

| 目录 | 用途 |
| --- | --- |
| `src/api` | 后端接口封装 |
| `src/views/modules` | 后台业务页面 |
| `src/router` | 基础静态路由 |
| `src/stores/modules/user.js` | 登录用户、权限列表、菜单 ID |
| `src/stores/modules/permission.js` | 动态路由生成 |
| `src/directive/permission` | `v-permission` 按钮权限 |
| `src/directive/rich` | 富文本展示前的 DOMPurify 清洗 |
| `.env.*` | `VITE_APP_*` 环境变量 |

## 动态菜单怎么来

登录后，`src/permission.js` 执行：

1. `getUserInfo()` 获取用户信息和是否管理员。
2. `listPermissions()` 获取按钮权限编码。
3. 非管理员调用 `listMenuIds()` 获取可见菜单 ID。
4. `permissionStore.generateRoutes(menuIds)` 调 `/mall4cloud_rbac/menu/route` 获取菜单路由。
5. `component` 为 `Layout` 时加载布局，否则从 `src/views/modules` 下按路径加载页面。

后台菜单的 `component` 字段必须和页面路径对应。例如 `product/category` 会加载 `src/views/modules/product/category/index.vue`。

## 按钮权限怎么来

页面按钮使用：

```vue
v-permission="['product:category:save']"
```

用户登录后，前端把 `/mall4cloud_rbac/menu_permission/list` 返回的权限编码放到 `roles`。`v-permission` 指令判断当前用户是否有对应编码，没有就移除按钮。管理员直接放行。

## 检查题

问题：新增菜单后，页面文件存在但点击菜单空白，最可能先查什么？

答案：先查 `menu.component` 是否能映射到 `src/views/modules/<component>/index.vue` 或 `src/views/modules/<component>.vue`。

为什么：动态路由通过 `import.meta.glob('../../views/modules/**/**.vue')` 加载页面，路径对不上就没有组件。
