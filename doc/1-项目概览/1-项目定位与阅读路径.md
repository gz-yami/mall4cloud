# 项目定位与阅读路径

Mall4cloud 开源版是一个微服务 B2B2C 商城系统。当前仓库包含后端微服务、数据库脚本、中间件配置和三个前端工程：平台端、商家端、uni-app 用户端。

## 适合谁读

| 读者 | 先解决的问题 |
| --- | --- |
| 新维护者 | 先把项目跑起来，再知道服务、数据库和前端怎么连起来。 |
| 二次开发者 | 先理解权限、数据表和后台管理功能模式，再新增后台功能。 |
| 开源评估者 | 先判断技术栈、运行成本、授权边界和是否符合自己的商城场景。 |

## 30 分钟目标：跑起基础链路

先不用跑全量业务，重点确认网关、登录、权限和一个后台端可用：

1. MySQL、Redis、Nacos、MinIO 先可用。
2. 网关 `mall4cloud-gateway` 在 `8000` 端口可访问。
3. 认证、权限、平台端或商家端相关服务启动。
4. 管理后台能打开登录页，并能请求网关。

如果使用 Docker Compose 中间件，先看 [中间件 Docker Compose](../8-部署运维/2-中间件-docker-compose/README.md)。

3. [前端工程与动态路由](../3-技术框架/2-前端工程与动态路由.md)

## 1 天目标：完成一次后台二开

优先做后台管理功能，不先碰订单、支付、库存、结算。可以从热搜、轮播图、附件分组这类后台 CRUD 学起，再迁移到自己的功能。

学习路线：

1. 读 [核心表速查](../7-数据模型/2-权限商品订单核心表.md)。
2. 读 [权限链路](../4-技术实现/1-登录认证与权限链路.md)。
3. 照 [后台管理功能二开示例](../5-二次开发/1-后台管理功能二开示例.md) 写自己的文档或代码。
4. 用 [命名约定与验收清单](../5-二次开发/2-命名约定与验收清单.md) 自检。

## 阅读原则

- 每个目录对应一个学习阶段。
- 检查题都给答案和原因。
- 结论以当前源码、配置和 SQL 为准。
