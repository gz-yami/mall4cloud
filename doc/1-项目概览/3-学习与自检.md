# 学习与自检

这部分说明怎么读文档、怎么确认自己读懂了。检查题不是考试，答案和原因都会直接给出。

## 阅读时先看三件事

| 问题 | 读到什么程度算够 |
| --- | --- |
| 我现在要做什么？ | 知道本页目标和下一步入口。 |
| 项目怎么实现？ | 能指出对应源码、配置或 SQL。 |
| 我怎么知道自己做对了？ | 能对照验收信号或检查题说明原因。 |

## 检查题怎么用

每道题都有三个部分：

1. 题目：先自己判断一次。
2. 答案：直接给出正确结论。
3. 为什么：说明依据来自哪个代码、配置或 SQL 事实。

示例：

问题：平台端前端应该配置 `VUE_APP_BASE_API` 还是 `VITE_APP_BASE_API`？

答案：配置 `VITE_APP_BASE_API`。

为什么：当前平台端和商家端是 Vite 工程，`.env.development` 中使用 `VITE_APP_BASE_API`，代码通过 `import.meta.env` 读取。

## 读者自检

| 阶段 | 自检答案应该是什么 |
| --- | --- |
| 启动后 | 网关 `8000` 可访问，后台登录页能请求到网关。 |
| 读完框架后 | 能说出 `Controller -> Service -> Mapper -> DB` 的基本链路和 Feign 的用途。 |
| 读完权限后 | 能说出菜单控制页面可见性，按钮权限控制按钮显示，后端接口权限控制请求是否放行。 |
| 读完数据模型后 | 能说出 `menu`、`menu_permission`、`role_menu`、`user_role` 的关系。 |

## 避免误判

- 不用截图代替源码、配置或 SQL 结论。
- 不把通用安装教程当成项目配置。
- 不把未经核对的命令写成当前可用。
